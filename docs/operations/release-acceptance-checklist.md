# 发布验收清单

本清单中的项目必须取得真实环境证据后才能标记完成。自动化测试通过不等同于 B 站、群晖或长期运行验证通过。

## 上线前真实验证

| 项目 | 前置条件 | 操作与预期 | 证据 | 失败回滚 |
| --- | --- | --- | --- | --- |
| 管理员登录 | 全新配置目录 | 完成首次设置，退出后重新登录；API Key 不再参与日常请求 | 登录页及会话接口截图 | 停止服务，保留 `auth.sqlite3` 后排查 |
| 投稿账号续期 | 两个扫码账号 | 连续运行并执行立即续期；账号保持可用且版本只在凭据更新时增加 | 账号页和续期日志 | 暂停账号写操作并重新扫码 |
| 投稿设置 | 测试房间及短录像 | 逐项验证标题、简介、分区、原创声明、仅自己可见、动态开关 | 创作中心稿件截图 | 将房间设为本场不上传 |
| 定时发布 | 投稿账号可用 | 选择未来时间投稿；稿件在约定时间发布 | 创作中心时间和视频链接 | 取消定时稿或删除测试稿 |
| 自定义封面 | 已上传手动封面 | 验证历史封面复用、直播封面和手动封面三种路径 | 稿件封面截图 | 改回直播封面 |
| 合集 | 账号已有或新建合集 | 创建合集、选择分节并投稿；审核后任务显示已加入 | 合集页面和任务详情 | 从 B 站后台移除测试视频 |
| 评论与弹幕 | 稿件审核通过 | 验证自动评论和各分 P 弹幕回灌；失败只在最终状态通知 | 视频页、CID 与任务进度 | 关闭房间对应功能并清理测试稿 |
| 转码自动修复 | 可构造转码失败样本 | 验证原文件重传、再次失败后 FFmpeg 流复制、仍失败后停止 | 修复阶段、FFmpeg 日志、同一 AID | 暂停任务并人工处理原稿 |
| 视频播放 | 本地大文件及已删本地文件 | Range 播放可快速起播；本地缺失时分 P 链接跳转 B 站 | 浏览器网络面板和链接 | 关闭播放抽屉，不修改录像 |
| 删除与重启恢复 | 正在上传及正在删除任务 | 中断服务后重启；安全阶段继续，未知远端结果保持暂停 | 重启前后数据库状态和日志 | 停止 worker，备份数据库后处理 |
| 容量与通知 | 设置容量及预警阈值 | 跨过阈值只通知一次，恢复后再通知一次；按房间策略清理 | 通知截图和保留/删除文件清单 | 改为从不删除并提高容量上限 |
| 群晖双网络 | Docker host 网络、多网关已启用 | 分别绑定录制、房间轮询、弹幕、上传、BILI API；断开主线路后切换 | 网络页、出口 IP、切换通知 | 全部用途恢复为系统默认网络 |

## 灰度与全量

1. 先接入 3～5 个房间连续运行至少 3 天，记录开播发现延迟、录制完整性、弹幕数量、上传速度、审核结果、账号和网络异常。
2. 灰度期间不得把“尚未验证”项目标记为完成；任何未知投稿结果都先暂停并核对创作中心。
3. 灰度无阻断问题后接入 58 个房间，观察至少一个完整直播高峰期，再确认批量房间刷新没有退化为逐房间高频请求。
4. 扩容前备份配置、`blrec.sqlite3`、`auth.sqlite3` 和凭据密钥；回滚时恢复上一镜像和对应数据库备份。
