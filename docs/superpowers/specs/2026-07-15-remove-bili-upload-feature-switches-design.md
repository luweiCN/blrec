# 删除投稿全局开关设计

## 目标

删除 `bili_upload` 下的 `enabled`、`auto_upload_enabled`、
`auto_comment_enabled` 和 `danmaku_backfill_enabled`。投稿账号管理作为应用固定能力启动，
不再因隐藏配置返回“未启用”。

## 行为

- 投稿账号、上传任务、投稿规则、封面和合集管理始终初始化；缺少凭据加密密钥等必要配置时，返回明确的配置错误。
- 是否上传仅由房间投稿规则及场次的“本场上传 / 本场不上传”决定。
- 是否生成自动评论、是否回灌弹幕仅使用上传任务冻结的投稿设置决定。
- 手动回灌不再受全局开关拦截，但仍要求稿件审核通过、账号有效、CID 与本地弹幕完整。
- 上传 worker 保持运行；没有符合条件的任务时不产生 B 站请求。

## 配置兼容

设置模型和前端类型删除四个字段。旧 TOML 中遗留字段按 Pydantic 现有的额外字段兼容行为忽略；设置下次保存时不再输出这些字段。数据库无需迁移。

## 验证

后端测试覆盖无开关时账号管理正常启动、上传任务正常领取、评论与弹幕按任务策略执行、手动回灌不被全局设置拒绝。前端设置模型和固件同步测试不得再包含这些字段，最后运行完整 Python 与 Angular 测试。
