# 多线路路由、可观测性与试运行设计

## 目标与边界

本轮解决多网卡自动发现、线路固定/轮换、故障接管和路由 DNS 不一致，使用统一 SSE 推送录制、上传和网络状态，提供 BLREC 自身流量统计与按网卡上传限速，补齐录制、上传、投稿及恢复流程的审计日志，并从旧 BLREC 迁移兼容设置和少量高频房间进行试运行。

轮换只分摊匿名监控流量。投稿账号、上传、评论、弹幕回灌等带账号凭据的请求始终固定线路，避免同一账号在多个公网出口间频繁跳变。Windows/macOS Docker Desktop 无法暴露的宿主机物理网卡不做虚假展示；Linux/群晖 host 网络和原生运行环境使用系统实际可见的网卡。

## 网卡发现与健康状态

平台提供器统一输出网卡名称、IPv4、掩码、网关、DNS、链路状态、速率和类型：

- Linux 通过 Netlink 读取全部路由表、策略规则和地址，覆盖群晖多网关；不再只依赖 `/proc/net/route`。
- Windows 原生使用 IP Helper 数据；macOS/BSD 读取系统路由信息。
- DNS 查询绑定网卡源地址，并由受路由管理的 aiohttp、requests 和上传协议共用解析结果，避免一条线路取得另一条线路的 Fake-IP。

没有外网路由的容器网桥（如 `docker0`）默认禁用，但允许用户手动启用。启用状态持久化；消失的网卡保留设置并显示不可用。探测分别核对网关、DNS、B 站 HTTPS 和公网出口 IP。

连续两次连接超时、DNS 失败或连接错误才判定线路不可用；HTTP 业务错误不计为线路故障。恢复探测成功后重新纳入可用集合。

## 实时状态、流量与限速

前端只建立一条 `/api/v1/realtime` SSE 连接，统一接收录制任务与房间状态、录制速率、上传阶段与进度、网卡流量、线路健康和需要重新同步的通知。页面首次打开仍通过 HTTP 读取完整快照；SSE 只推送变化。连接建立或断线重连后，服务端发送 `resync`，前端重新读取当前页面快照，不依赖事件重放恢复状态。

SSE 每 15 秒发送心跳，响应设置 `Cache-Control: no-cache` 和 `X-Accel-Buffering: no`。HTTPS 与 HTTP 使用同一路径和当前页面协议；TLS 不作为关闭 SSE 的理由。每个订阅者使用有界队列，积压时丢弃增量并发送 `resync`，避免慢客户端拖垮服务端。

录制任务页面现有每秒 HTTP 轮询改为一次快照加 SSE 增量。服务端每秒采样任务和网卡指标，只推送已变化的数据。上传进度以 B 站已确认分片为准，从持久化分片大小与状态计算已确认字节、总字节、百分比、速度、预计剩余时间及当前分 P；重启后可从数据库恢复准确进度，速度在收到新样本前显示为空。

host 网络下的系统网卡计数包含 NAS 其他进程，不能代表当前容器。因此流量计量放在 BLREC 自己的 HTTP、WebSocket、录制流和上传流入口，按网卡和用途累计有效载荷字节；不包含 TCP/TLS 开销，并在界面明确说明。每块网卡显示当前上传/下载速度和累计字节。

每块网卡可设置共享上传限速，`0` 表示不限速。限速器采用按网卡共享的令牌桶，将 UPOS 请求体拆成小块平滑发送，并保留正确的 Content-Length；修改后立即影响后续数据块。下载不提供限速。禁用网卡会立即阻止新连接和新分配；正在上传的任务在安全分片边界暂停，现有录制与 WebSocket 不被强制切断。

## 线路策略

每种网络用途选择“固定线路”或“轮换线路”，并可开启自动接管：

- 房间状态轮询：固定时每批使用同一线路；轮换时按批次分配。
- 弹幕 WebSocket：建连时分配，连接和普通重连保持不变；线路不可用且开启接管时才换线。
- 录像下载：每场直播开始时分配，整场及其分段保持不变；接管会创建新的录像分段。
- 视频上传：只允许固定线路；故障时暂停并报错，不自动换线。
- 匿名 B 站 HTTP：可按完整请求轮换。
- 投稿账号请求：只允许固定线路，包括投稿、审核、评论、回灌和账号维护。

轮换使用已启用且健康的线路，维护稳定游标，不在单个业务会话内部切换。固定线路开启接管后，从其余健康线路中稳定选择下一条；全部不可用时返回明确错误，不静默使用系统默认线路。旧 `primaryInterface/fallbackInterface` 配置迁移为固定线路和自动接管，不再要求用户指定备用线路。

## 网络页面交互

“网卡与出口”模块标题右侧只保留“检测全部线路”。每行显示网卡、地址、网关、公网出口、健康状态、实时上传/下载速度、累计流量、启用开关、上传限速和“检测”。启停与限速使用行内确认并立即调用接口持久化，不设置模块级保存按钮；行内“检测”只探测该网卡，并仅让该行显示加载状态。

“网络分工”模块标题右侧放置唯一的“保存网络分工”按钮。固定/轮换、线路和自动接管的修改统一由该按钮保存；页面顶栏不再放保存按钮。

## 日志与投稿回读

沿用 Loguru 文件日志，每天零点轮换，默认保留 60 天。INFO 记录关键状态变化，DEBUG 记录诊断细节。统一记录事件名、用途、房间/场次/任务、账号 ID、线路、源地址、选择原因、尝试次数、耗时和结果；Cookie、token、API Key、密码及完整请求体禁止入日志。

审计范围包括设置修改、任务增删启停、线路选择/轮换/接管/恢复、流量限速修改、录制与分段、文件处理、上传分片摘要与每约 5% 的进度、投稿、审核、转码修复、评论、弹幕回灌、文件删除和重启恢复。分片完成写 DEBUG，进度里程碑写 INFO。

稿件审核完成后使用实际投稿账号读取远端稿件详情，将任务快照与可回读的标题、简介、分区、标签、创作声明、可见性、封面、合集、定时发布、评论/弹幕开关及分 P 逐项比较。数据库和任务详情保存“通过、存在差异、部分无法验证、验证失败”及检查时间；无法由接口证明的字段明确标记，不视为成功，也不自动重复投稿。

## 旧配置迁移与试运行

迁移前备份新版设置和数据库。旧版在兼容字段上优先，迁移输出模板与分段限制、UA、弹幕、录像、封面和后处理设置；日志保留期改为 60 天。保留新版的网络、批量监控、账号、上传数据库、通知路由、容量和删除策略。不迁移浏览器 Cookie、旧 Webhook、废弃磁盘检测字段及通知密钥。

## 验收

- 群晖显示 `ovs_eth0/192.168.1.1` 与 `ovs_eth1/192.168.50.1`，`docker0` 默认禁用，两条真实线路均可独立完成 B 站探测。
- 单元测试覆盖策略路由、源地址 DNS、固定/轮换粘性、接管、全部不可用、流量计量、共享上传限速、SSE 重连同步和旧配置迁移；Linux 容器执行集成测试。
- 通过模拟断线确认上传不换线、录像按场次粘住、WebSocket 按连接粘住、状态批次轮换。
- 录制任务不再每秒发起 HTTP 轮询；一条 SSE 能同时更新录制状态、上传进度和网卡速率，断线后自动恢复快照。
- 上传限速在并发任务下仍遵守网卡总上限，且不会限制录像下载。
- NAS 连续运行 3～5 天后，可仅凭数据库状态和日志还原每次关键决策，并核对配置是否真正生效。
