# 上传任务导航拆分设计

## 目标

将当前嵌在“投稿账号”页面中的录制会话列表迁移为独立的顶层“上传任务”页面。投稿账号页面只保留账号登录、续期、主账号及关联关系管理。

## 方案

采用新的懒加载路由 `/upload-tasks`，保留现有 `/uploads` 作为投稿账号地址。相比重命名现有路由或在同一页面增加二级 Tab，这种方式改动最小，不会破坏设置页中已有的账号管理链接，也能让两个业务入口在侧边栏处于同一层级。

新增 `UploadTasksModule` 和页面壳组件，录制会话组件及其模型、服务整体迁入该模块。页面标题改为“上传任务”，列表标题改为“上传任务列表”。本次只调整页面归属，不伪造尚未实现的上传、投稿审核或弹幕回灌状态；当前行继续展示真实的录制会话和分 P 制品状态。

## 导航与交互

侧边栏顺序为“任务”“上传任务”“投稿账号”“设置”“关于”。“上传任务”使用独立链接和折叠态提示。页面保留现有刷新、加载、失败重试、空状态、降级提示和可访问表格结构。

## 验证

- 导航测试验证 `/upload-tasks` 与 `/uploads` 分别显示正确名称。
- 投稿账号组件测试验证不再渲染录制会话组件。
- 上传任务页面测试验证页面标题和列表组件存在。
- 录制会话组件原有测试在新目录继续通过。
- 完整 Angular 测试、ESLint 与生产构建通过。

## 后续边界

拆分完成后，下一步扩充录制场次和分 P 的上传必需元数据，再由已有 `upload_jobs`、`upload_parts` 表建立真实上传任务状态机。本次不创建虚假任务，也不执行 B 站写操作。
