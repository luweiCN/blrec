# 线上录制、媒体预览与管理界面回归修复设计

## 范围与目标

本轮只修复 NAS 验证中已经复现的四类回归：直播已确认但未开始录制、播放与高光剪辑停在加载状态、浏览器插件连接成功后仍显示连接表单，以及网络管理表格无谓横向滚动。修复后发布新的测试版并更新 NAS，不扩展新的业务功能。

## 弹幕连接与录制解耦

弹幕协议所需的短期 token 每次连接前重新获取；它与投稿账号 Cookie 是两种凭据。每一场直播都从匿名模式开始：使用同一条匿名网络会话、同一套无 Cookie 请求头依次获取 `getDanmuInfo` token 并建立 WebSocket，认证身份为 `uid=0`。匿名连接经过现有有界重试仍失败时，只选择一次当前可用的读取账号凭据，并冻结该凭据版本，使 token 请求、WebSocket 请求头、UID 与 BUVID 始终来自同一身份。

Cookie 兜底成功后，本场后续重连保持 Cookie 模式；兜底失败后不轮换更多账号规避限制，只按现有退避记录失败。房间下播时清除本场连接模式，下一次开播重新匿名优先。日志记录房间、连接模式、阶段、网络线路、尝试次数和脱敏错误，不记录 token 或 Cookie。

直播状态一经批量监控确认，视频录制立即启动，不能等待弹幕 WebSocket。弹幕连接在独立任务中建立和重试；失败只影响相应时间段的弹幕，不停止视频录制。下播时同时结束录像并取消弹幕连接任务。

## 前端加载与插件连接状态

SSE 事件统一回到 Angular zone 后再发布，避免由 `resync` 发起的 HTTP 回调更新了组件状态却不刷新页面。播放弹窗和高光编辑器在媒体授权、时间轴和播放器错误回调完成时显式刷新自身视图，覆盖弹窗 overlay 与初次路由加载场景。

插件选项页启动时验证已保存授权。有效时隐藏地址、用户名和授权表单，只显示“已连接”、后端地址、管理员用户名和“修改”按钮；修改时回填原值，旧授权保留到新连接成功。授权失效时自动展示表单并提示重新连接。

## 网络表格布局

网络面板不再额外创建横向滚动层，表格列宽缩减为当前内容所需宽度，操作列始终在桌面页面内可见。只有视口确实小于表格最小可读宽度时，才由表格自身提供单层横向滚动。

## 验证

后端测试覆盖匿名成功不读 Cookie、匿名失败后单次 Cookie 兜底、同场保持模式、下播重置以及弹幕失败不阻塞录制。前端测试覆盖 SSE zone、异步媒体回调、插件已连接/失效/修改状态和网络表格宽度。发布后在 NAS 上确认当前直播房间产生录像文件、插件出现“添加高光”，并实际打开本地播放与剪辑页面。
