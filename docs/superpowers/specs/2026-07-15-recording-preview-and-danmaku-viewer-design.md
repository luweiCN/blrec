# 录播预览与弹幕查看设计

## 目标与范围

在上传任务的分 P 详情中分别提供本地录像播放和已保存弹幕查看。已完成文件正常播放；仍在写入的 FLV 采用“快照播放”：打开播放器时固定当时已经写入的字节、时长和关键帧索引，可在该范围内拖动。重新打开播放器后才读取新增内容。本期不做持续追播、实时弹幕刷新、弹幕覆盖层、服务端转码或 HLS。

## 页面交互

分 P 表格的“文件”列提供“播放”，只在本地视频存在时打开独立播放器弹窗；“弹幕”列提供“查看”，打开独立弹幕弹窗。播放器里不再放弹幕标签，弹幕弹窗里也不放视频。录制中的 FLV 使用“播放已录内容”文案。

稿件审核通过后，分 P 名称显示为蓝色 B 站链接；它与本地“播放”同时存在，不能因为本地文件仍在就隐藏。缺少本地文件时不再先打开播放器弹窗做跳转，用户直接点击分 P 链接。稿件尚不可见时不提供远端链接。

弹幕列表按录制时间排序，每页 100 条，展示视频时间、用户名、UID 和内容。XML 缺少用户名或 UID 时显示“未记录”，不猜测用户身份。录制中的弹幕 XML 尚未登记到场次时，弹幕操作等文件完成后再出现，本期不读取正在增长的 XML。

录制中的文件显示固定提示：“当前可播放至打开播放器时的录制进度；重新打开可读取新增内容。”播放器不自动重新连接。播放失败时区分文件不存在、索引不可用、编码不受浏览器支持和网络读取失败，不以无限重试掩盖错误。

## 媒体接口

后端按录制分 P ID 读取媒体，不接收客户端文件路径。接口优先读取存在的成品文件，否则读取原始录制文件。创建录制中 FLV 的播放凭据时，同时固定当前文件大小、录制器实时统计的时长和不超过该文件大小的关键帧索引，形成有有效期的播放快照。

播放凭据返回快照标识、毫秒时长和逻辑文件大小，前端据此把 FLV 作为有限文件交给播放器。服务端只在有上限、短时有效的快照缓存中保存虚拟前缀和关键帧索引，不缓存视频字节；过期后提示用户重新打开播放器。应用重启会使旧快照失效，不影响原录像。

原始直播 FLV 的首段元数据把时长写为 0，且没有可供随机读取的关键帧表。服务端为快照生成仅包含 FLV 头和更新后 `onMetaData` 的虚拟前缀，并把后续逻辑字节范围映射回原文件；不改写录制文件、不复制视频主体，也不调用 FFmpeg。元数据中的关键帧位置按虚拟前缀长度修正。

接口支持单段 HTTP Range 请求，返回 `Accept-Ranges`、`Content-Length` 和必要的 `Content-Range`；无效范围返回 416。每个快照的逻辑大小及索引在有效期内保持不变，即使录制器继续写入也不读取快照末尾之后的字节。文件消失或数据库记录不存在返回 404，路径不是普通文件或不属于该分 P 时拒绝访问。

MP4 使用浏览器原生 `<video>`。FLV 使用固定的 `mpegts.js 1.8.0` 在浏览器内将 H.264/AAC 数据封装为 Media Source 可播放片段。录制中快照按有限文件加载，并显式提供毫秒时长和逻辑文件大小；播放器读取虚拟元数据中的关键帧表，通过 Range 请求完成未缓冲位置的跳转。浏览器不支持 Media Source、实时索引暂时不可用或文件编码不兼容时显示明确提示，并降级为从头顺序播放，不自动转码或扫描整个大文件。

远端观看不代理或解析 B 站视频流，也不新增 B 站查询请求。前端使用任务中已经保存并经过审核核对的 BVID，以及数据库中的 `part_index`，生成 `https://www.bilibili.com/video/{bvid}?p={part_index}`，在新标签页打开并设置 `rel="noopener noreferrer"`。因此不会增加后台轮询频率，也不把用户观看流量转发到 NAS。

## 弹幕接口

后端新增按分 P ID 分页读取 XML 的认证接口。使用禁用实体与网络访问的流式解析，只处理普通弹幕 `<d>`；礼物、上舰和 SC 不是用户普通弹幕，本列表暂不混入。响应包含原始序号、相对视频时间、用户名、UID、正文及下一页游标。

接口只读取场次已经登记且存在的完整 XML。翻页使用游标，不把整份 XML 或任意本地路径发送给浏览器。本期不增加全文搜索或额外弹幕索引；如果真实大文件的翻页性能不足，再基于数据补充按需索引。

## 前端边界与清理

播放器和弹幕查看器使用两个独立组件，上传任务列表只负责传入分 P ID 和展示对应弹窗，不保留“当前标签”之类的共享状态。关闭播放器、切换分 P 或重新加载时必须销毁播放器实例、取消未完成请求并释放 Media Source，避免长时间使用页面时积累连接和内存。

弹幕内容按普通文本渲染，不使用 HTML。媒体及弹幕接口继续使用现有 API Key 鉴权；日志不得输出 Cookie、API Key 或完整弹幕正文。

## 验证

后端测试覆盖完整文件、实时元数据与当前分 P 的匹配、关键帧越界过滤、虚拟前缀偏移修正、录制中增长文件的固定快照边界、跨前缀 Range、普通与后缀 Range、416、快照过期、文件缺失、越权路径、弹幕分页游标以及 XML 实体攻击。前端测试覆盖两个入口打开不同弹窗、播放器中没有弹幕标签、MP4/FLV 播放器选择、快照时长、未缓冲位置拖动、重新打开、错误降级、播放器销毁、弹幕分页和文本安全。

浏览器验收使用当前 H.264/AAC 录制中 FLV：确认显示打开时的有限总时长，可拖动到尚未缓冲的中间位置并由 Range 请求快速起播，不能越过快照末尾，关闭后重新打开可读取更多内容；再用已完成文件验证正常播放和弹幕翻页。监控录制器的写入速率与日志，确认只读预览不会暂停录制、复制大文件或持续占用 NAS CPU。
