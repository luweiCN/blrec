# 录播预览与弹幕查看设计

## 目标与范围

在上传任务的分 P 详情中直接查看已保存的弹幕，并播放录制文件。已完成文件正常播放；仍在写入的 FLV 采用“快照播放”：打开播放器时只读取当时已经写入的字节，播放到该位置即停止，点击“重新加载最新内容”后才读取新增内容。本期不做持续追播、实时弹幕刷新、弹幕覆盖层、服务端转码或 HLS。

## 页面交互

分 P 表格增加“播放录像”和“查看弹幕”操作。录制中的 FLV 使用“播放已录内容”文案。存在本地视频时始终优先播放本地文件；本地视频已经删除、稿件已经审核通过且保存了 BVID 时，改为显示“在 B 站观看”，打开该分 P 对应的 B 站页面。缺少本地视频且稿件尚不可见时不显示播放操作，并明确展示“本地录像已删除”。

两个操作打开同一个大尺寸内容弹窗，并分别聚焦视频或弹幕区域。桌面端左侧为 16:9 播放器，右侧为弹幕列表；窄屏改为上下排列。弹幕列表按录制时间排序，每页 100 条，展示视频时间、用户名、UID 和内容。XML 缺少用户名或 UID 时显示“未记录”，不猜测用户身份。录制中的弹幕 XML 尚未登记到场次时，弹幕操作等文件完成后再出现，本期不读取正在增长的 XML。

录制中的文件显示固定提示：“播放的是打开时已经写入的内容；重新加载后可读取新增内容。文件完成前可能无法拖动。”播放器不自动重新连接。播放失败时区分文件不存在、编码不受浏览器支持和网络读取失败，不以无限重试掩盖错误。

## 媒体接口

后端新增按录制分 P ID 读取媒体的认证接口，不接收客户端文件路径。接口优先读取存在的成品文件，否则读取原始录制文件，并在打开响应时固定当前文件大小。

接口支持单段 HTTP Range 请求，返回 `Accept-Ranges`、`Content-Length` 和必要的 `Content-Range`；无效范围返回 416。读取过程最多到本次响应捕获的文件末尾，即使录制器继续写入也不等待新字节。文件消失或数据库记录不存在返回 404，路径不是普通文件或不属于该分 P 时拒绝访问。

MP4 使用浏览器原生 `<video>`。FLV 使用固定的 `mpegts.js 1.8.0` 在浏览器内将 H.264/AAC 数据封装为 Media Source 可播放片段，不修改原文件，也不占用 NAS 转码资源。浏览器不支持 Media Source 或文件编码不兼容时显示明确提示，本期不自动转码。

远端观看不代理或解析 B 站视频流，也不新增 B 站查询请求。前端使用任务中已经保存并经过审核核对的 BVID，以及数据库中的 `part_index`，生成 `https://www.bilibili.com/video/{bvid}?p={part_index}`，在新标签页打开并设置 `rel="noopener noreferrer"`。因此不会增加后台轮询频率，也不把用户观看流量转发到 NAS。

## 弹幕接口

后端新增按分 P ID 分页读取 XML 的认证接口。使用禁用实体与网络访问的流式解析，只处理普通弹幕 `<d>`；礼物、上舰和 SC 不是用户普通弹幕，本列表暂不混入。响应包含原始序号、相对视频时间、用户名、UID、正文及下一页游标。

接口只读取场次已经登记且存在的完整 XML。翻页使用游标，不把整份 XML 或任意本地路径发送给浏览器。本期不增加全文搜索或额外弹幕索引；如果真实大文件的翻页性能不足，再基于数据补充按需索引。

## 前端边界与清理

播放器和弹幕请求封装在独立的分 P 内容组件中，上传任务列表只负责传入分 P ID 和展示弹窗。关闭弹窗、切换分 P 或重新加载时必须销毁播放器实例、取消未完成请求并释放 Media Source，避免长时间使用页面时积累连接和内存。

弹幕内容按普通文本渲染，不使用 HTML。媒体及弹幕接口继续使用现有 API Key 鉴权；日志不得输出 Cookie、API Key 或完整弹幕正文。

## 验证

后端测试覆盖完整文件、录制中增长文件的快照边界、普通与后缀 Range、416、文件缺失、越权路径、弹幕分页游标以及 XML 实体攻击。前端测试覆盖操作文案、MP4/FLV 播放器选择、重新加载、错误提示、播放器销毁、弹幕分页、文本安全，以及本地文件存在时优先播放、本地文件缺失时生成正确分 P 链接、稿件未通过审核时不提供无效远端入口。

浏览器验收使用当前 H.264/AAC 录制中 FLV：确认可从头顺序播放，播放请求不会等待文件继续增长，重新加载后可读取更多内容；再用已完成文件验证正常播放和弹幕翻页。现有上传、投稿和录制流程不得因只读预览受到影响。
