# 上传任务控制与恢复补充设计

## 范围

本设计补充现有“录制场次与上传任务生命周期”方案，并以本轮最新决定为准：投稿规则在上传任务实际创建时读取，因此录制期间修改规则会作用于尚未创建的任务；不会追溯自动创建历史任务。

## 暂停、继续与修改

上传任务增加“暂停上传”和“继续上传”。暂停先持久化管理员暂停标记，再让分片上传在安全检查点释放租约；已经确认的分片不清空，未知结果不盲重试。继续只清除管理员暂停标记并从持久化进度恢复。

“修改任务”在一个弹窗中修改投稿账号和本场投稿设置。只有任务尚未产生任何远端副作用、没有活动租约、所有分 P 仍为 `prepared` 且投稿为 `prepared` 时允许修改。换账号后重新校验分区、创作声明、封面和合集；不属于新账号的合集自动清空并要求用户确认。任务开始上传后页面说明不可修改的原因。

## 删除与重启恢复

任何场次都可以删除。删除请求先写入 `requested` 终止标记，所有录制、后处理、上传、审核、修复、合集、评论和弹幕 worker 领取工作前均跳过该场次。执行器依次停止活动工作、幂等删除本地视频/弹幕/封面/临时文件和本地数据库记录；永远不删除 B 站稿件。

删除阶段和错误持久化。进程重启后自动继续 `requested/deleting/failed` 删除任务；不存在的文件视为已删除。既有上传分片、投稿、审核和修复沿用租约恢复，管理员暂停任务保持暂停。录制器重启后继续以新分段写入同一可恢复场次，无法复用时创建同场续录分段，不能要求人工确认。

## 状态详情与后台失败

任务详情展示合集状态/失败原因、定时发布时间/发布状态，以及自动评论和弹幕回灌的最终失败原因。评论、弹幕、合集的瞬时错误自动退避；最终失败保留在任务详情并触发通知，不提供人工重试按钮。

列表操作只保留真实可用项：本场上传/不上传、创建任务、暂停/继续、修改任务、重试失败上传、重新投稿、回灌尚未启用的弹幕、转码修复和删除。没有“立即检查审核”“重新扫描文件”“编辑被拒原 AID”“人工重试评论/弹幕”等本轮未确认操作。

## 验证

后端测试覆盖安全修改边界、账号/合集校验、暂停检查点、继续保留进度、所有状态删除、活动 worker 终止、删除重启续跑和外部稿件不删除。前端测试覆盖能力驱动菜单、修改弹窗、暂停/继续反馈、详情状态和简洁删除确认。重启验收检查录制与上传均能继续，且不会重复投稿。
