# 上传任务操作与转码修复设计

## 目标与原则

上传任务列表提供按状态可用的单项和批量操作，让失败任务可以人工立即恢复，同时保持自动状态机的幂等与限速。操作不直接暴露数据库状态，不清空已经确认的上传结果，也不通过新建稿件来“修复”原稿。

Java 参考项目中的“触发发布事件”和“转码失败修复”保留其有效语义；“重置为未上传发布”“强制结束录制状态”和删除整条历史记录不照搬，因为这些操作可能产生重复稿件、破坏活动录制或丢失弹幕。

## 单项操作

列表行使用一个“操作”下拉菜单，只展示当前状态能够安全执行的命令：

- **创建上传任务**：场次已经结束、尚无任务、房间投稿规则有效且本地视频存在时可用。
- **重试失败步骤**：恢复失败或暂停的上传、投稿或审核同步，从已持久化的阶段继续；已确认分 P 和已保存 AID/BVID 不重置。
- **修复转码失败分 P**：稿件已有 AID/BVID，B 站明确报告分 P 缺失或转码终止失败，且本地视频仍存在时可用。
- **重新提交未通过稿件**：编辑原 AID，不创建新稿件。投稿账号保持任务原账号；标题、简介、分区、标签、声明、封面和发布设置重新读取该房间当前规则，账号不兼容的合集选择不沿用。
- **重试自动评论**、**重试弹幕回灌**：只恢复确认失败或安全暂停的项目，保留已完成项。结果未知的评论或弹幕必须继续使用现有逐条核对流程，不参与批量重发。
- **删除本地视频**：仅在录像已经停止且文件未被后处理或上传持有时可用；调用统一录像清理器，只删除视频并保留任务、XML、封面和数据库记录。

API 返回每个任务当前可用的操作代码和不可用原因，前端不自行复制状态判断。活动录制、活动上传、账号失效、缺少本地文件和结果未知等情况显示明确原因，不提供无效按钮。

## 批量操作

表格增加复选框；选择当前页任务后，在列表上方显示批量操作栏。首版支持当前页最多 100 条，不跨分页保留选择，避免用户看不到实际目标。

批量操作与单项操作使用同一组命令：创建任务、重试失败步骤、修复转码失败分 P、重新提交未通过稿件、重试评论、重试弹幕回灌和删除本地视频。后端逐条重新校验资格，不要求所选任务状态完全相同；不适用的任务记为“跳过”，响应汇总已入队、跳过和失败数量，并给出逐项原因。

批量操作只写入持久化操作队列，不在一个 HTTP 请求中同时调用 B 站。后台按现有账号限速和并发控制逐个处理；同一任务同一类型只能存在一个活动操作，重复点击返回已经入队。删除视频和重新提交前显示目标数量与影响说明并二次确认。

## 两阶段转码修复

修复开始前使用任务原账号查询原稿分 P 状态，并按已经核对的 `part_index`、CID 和远端标识匹配。只有 B 站连续检查后明确进入终止失败，或原稿中确认缺少该分 P，才开始修复；仍在排队或转码中的分 P 继续等待，不沿用 Java 项目把处理中状态过早视为失败的逻辑。

第一阶段重新上传原本地视频，获得新的远端文件标识后调用编辑稿件接口替换原 AID 中的失败分 P。其他正常分 P 保持原远端标识和顺序。提交后重新进入审核/转码观察，不创建第二份稿件。

同一分 P 再次被 B 站明确判定转码失败时进入第二阶段：使用 FFmpeg 以流复制方式生成临时修复文件，重新封装容器并规范时间戳，不重新编码、不改变清晰度。生成后用 ffprobe 校验视频流、音频流、时长和可读性；校验失败则停止。校验通过后上传修复文件，并再次编辑同一 AID。

每个分 P 最多执行一次原文件重传和一次重新封装修复。第二阶段仍失败时标记“自动修复失败”，保存 B 站状态码、阶段和本地检测摘要供页面查看，不继续循环，也不默认执行消耗 NAS CPU 且可能损失画质的完整转码。临时修复文件在操作结束后清理，原录像仍遵循房间保留策略。

## 持久化、并发与审计

操作队列记录任务、操作类型、发起人、状态、阶段、尝试次数、结果摘要和时间。进程重启后继续未完成操作。执行前通过条件更新取得操作租约，并再次确认任务没有活动上传租约；任务状态已经前进时将过期操作幂等跳过。

所有操作复用任务冻结账号或明确规定的原稿账号，不回退到其他账号。日志和审计记录操作对象与状态变化，不记录 Cookie、CSRF 或视频内容。批量命令不会绕过账号风控退避、请求间隔或 B 站返回的频率控制。

## 页面反馈

任务主状态保持面向用户的一行展示。操作入队后显示简短进度，例如“等待修复”“重传原文件”“重新封装修复”“等待 B 站转码”和“修复失败”；详情抽屉展示各失败分 P 的阶段与原因。批量完成提示汇总结果，跳过项可展开查看，不因某一条失败阻断其他任务。

## 验证

后端测试覆盖操作资格、重复入队、活动租约保护、失败阶段续跑、进程重启恢复、批量部分成功和审计记录。转码修复测试覆盖仍在转码时不误触发、仅替换失败分 P、沿用原 AID、原文件重传成功、重新封装修复成功、ffprobe 校验失败、两阶段耗尽和临时文件清理。

前端测试覆盖状态相关菜单、不可用原因、当前页多选、危险操作确认、批量汇总和修复阶段展示。真实验收先用测试任务验证失败重试与原稿编辑；只有出现真实 B 站转码失败稿件时才执行修复验收，并核对没有生成重复稿件、正常分 P 未被替换。
