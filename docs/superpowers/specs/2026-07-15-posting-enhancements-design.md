# 投稿增强设计

## 目标与范围

在现有真实上传与投稿链路上补齐合集、自定义封面和 B 站定时发布。配置继续归属于每个录制房间，并在创建上传任务时冻结，避免房间设置或主账号后来变化影响已经产生的任务。本期不做合集重命名、删除、自定义小节管理，也不做自动截帧封面。

## 合集

合集属于投稿账号。房间选择固定投稿账号时，展示该账号的合集与小节；选择“跟随主账号”时，根据当前解析出的主账号加载，主账号变化后清空房间原有合集选择，要求重新确认，避免把另一个账号的 ID 错当成同一合集。

选择器按“合集 / 小节”展示现有数据，并允许在弹窗内创建合集。创建时只填写标题和简介，调用 B 站接口创建合集；B 站生成默认“正片”小节后刷新列表并自动选中。上传任务保存投稿账号 ID、合集 ID 和小节 ID 的快照。

投稿审核通过且各分 P 的 CID 已核对后，再把稿件加入冻结的小节。加入失败作为独立后处理状态重试并展示原因，不把已经成功的稿件标成投稿失败。请求始终使用该任务冻结的投稿账号，不回退到其他可用账号。

## 封面

房间只有两种封面来源：使用该场直播的封面，或使用房间选定的手动上传封面。默认使用直播封面；优先读取录制时已保存的本地封面，否则根据该场次冻结的 `cover_url` 下载一次。无法取得选定封面时保留明确错误并重试，不静默换成其他房间或其他历史封面。

手动上传的封面进入本地封面库，按文件内容哈希去重，保存原图和缩略图。封面库只收录用户手动上传的图片，不保存每场直播的封面。房间可从图库快捷选择，也可上传后立即选中；任务冻结具体封面资源版本，之后替换房间封面不会改变旧任务。

图片在本地完成格式、尺寸与文件大小校验。向 B 站投稿前，使用任务账号上传封面，并按“本地封面资源 + 投稿账号”缓存返回的远端地址，避免同一账号重复上传同一张固定封面；不同账号不共用远端地址。封面读取接口只接受资源 ID，不接受任意文件路径。

## 定时发布

房间投稿设置提供“立即发布”和“投稿后延迟 X 小时公开”。延迟范围按 B 站接口限制校验为 2 小时至 15 天。提交稿件时计算绝对 Unix 时间并写入 B 站投稿参数 `dtime`，由 B 站负责定时公开；系统不自建定时投稿队列。

任务保存实际计划公开时间。B 站返回定时发布状态时，页面显示“等待定时发布”，审核轮询继续按正常低频节奏运行；稿件公开并核对分 P 后，再执行加入合集、自动评论和弹幕回灌。立即发布不发送 `dtime`。非法延迟阻止保存或投稿，不静默改成立即发布。

## 安全与失败处理

所有合集和封面请求使用任务冻结账号的有效凭据。接口日志不记录 Cookie、CSRF 或图片内容。网络错误使用现有退避与暂停机制；账号失效时暂停该任务并要求恢复原账号，不自动切换投稿身份。合集失败、评论失败和弹幕回灌失败各自重试，不重复创建稿件。

## 验证

后端测试覆盖账号隔离、主账号变化清空合集、创建后自动选择、审核后加入合集、独立失败重试、封面哈希去重、按账号缓存、直播封面回退下载、定时范围和 `dtime` 计算，以及定时状态到公开状态的恢复。前端测试覆盖合集级联选择与创建、封面图库、封面预览、发布方式和错误提示。

真实验收使用测试账号分别验证立即与定时投稿、直播封面与固定封面、创建合集并加入正确小节；随后在 B 站创作中心或稿件页面核对标题、封面、公开时间、账号和合集归属。
