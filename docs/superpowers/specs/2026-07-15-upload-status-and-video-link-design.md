# 上传任务状态与稿件链接设计

## 目标

上传任务列表只展示一个面向用户的投稿进度，不再同时显示任务状态和“投稿：已确认”等内部子状态。审核完成后，直播标题直接链接到对应的 B 站稿件。

## 状态展示

没有上传任务时，正在录制的场次显示“等待录制结束”，已经结束但尚未创建上传任务的场次显示“未创建任务”。存在上传任务时按任务状态显示：

- `waiting_artifacts`：等待文件处理
- `ready`：等待上传
- `uploading`：正在上传
- `submitting`：正在投稿
- `waiting_review`：等待审核
- `approved`、`completed`：投稿完成
- `rejected`：审核未通过
- `paused`：处理暂停

列表删除 `submit_state` 的第二行展示。详情抽屉也只展示同一主状态；自动评论与弹幕回灌继续作为独立后续处理结果展示。投稿结果未知、审核驳回或其他异常通过“处理暂停”或“审核未通过”以及具体原因表达，不再要求用户理解内部状态机。

## 稿件链接

当任务状态为 `approved` 或 `completed` 且已经保存 BVID 时，列表和详情中的直播标题显示为蓝色链接，目标为 `https://www.bilibili.com/video/{bvid}`，在新标签页打开并设置 `rel="noopener noreferrer"`。其他状态、缺少 BVID 或尚未通过审核时仍显示普通文本，避免把尚不可访问的稿件误示为完成。

## 验证

前端测试覆盖以下场景：

1. 列表只出现一个投稿状态，不再出现“投稿：已确认”。
2. `approved` 和 `completed` 统一显示“投稿完成”。
3. 已完成且有 BVID 的标题生成正确、安全的新标签链接。
4. 等待审核、缺少 BVID 及未创建任务的标题保持普通文本。
5. 刷新、分页和打开详情的现有交互不受影响。
