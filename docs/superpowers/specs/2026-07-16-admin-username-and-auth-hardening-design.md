# 管理员用户名与低成本登录防护设计

## 目标

在现有单管理员会话认证上增加一个由部署环境指定的管理员用户名，并补齐初始化、密码恢复和反向代理场景下的限速保护。保持单账号、单密码、无验证码和无双因素认证，不引入 Redis、外部身份服务或新的运维组件。

## 配置与兼容性

- 新增 `BLREC_ADMIN_USERNAME`，作为唯一管理员用户名的权威来源。群晖 Docker 部署必须显式配置；普通命令行启动未配置时使用 `admin`，兼容现有用法。
- 继续使用 `BLREC_API_KEY` 作为初始化与密码恢复安全码。网页只提交它进行即时校验，不保存到浏览器或认证数据库。
- 群晖 Docker 配置继续强制提供 `BLREC_API_KEY`。普通命令行未配置时保留现有的“仅本机回环地址可初始化或恢复”行为。
- 新增可选的 `BLREC_FORWARDED_ALLOW_IPS`。默认不信任任意代理转发的客户端地址；使用群晖反向代理时只填写实际代理地址或受控网段。
- 用户名不写入认证数据库。修改环境变量并重启后，新的用户名立即生效，已有密码和会话数据无需迁移。

## 交互流程

首次设置页面要求输入管理员用户名、初始化安全码和新密码。用户名必须与 `BLREC_ADMIN_USERNAME` 一致，安全码必须与 `BLREC_API_KEY` 一致，二者均正确后才保存 Argon2id 密码哈希并创建会话。用户名长度为 1～64 个可见字符，区分大小写，并拒绝首尾空白和控制字符。

日常登录只要求管理员用户名和密码，不再显示或提交初始化安全码。修改密码仍要求当前会话、当前密码和新密码。

忘记密码时要求管理员用户名、初始化安全码和新密码。用户名与安全码均正确后替换密码，并撤销全部现有会话。

所有凭据错误统一返回“用户名或密码错误”或“初始化凭据错误”，不向未认证客户端指出具体错误字段。用户名允许浏览器正常记忆；密码使用标准自动填充属性；初始化安全码始终关闭保存和自动填充。

## 防暴力破解

- 保留现有登录保护：同一客户端五分钟内失败五次，锁定十五分钟，并返回 `429` 与 `Retry-After`。
- 初始化和密码恢复共用一组独立的客户端限速，避免安全码被无限尝试；普通登录成功不会清除初始化凭据的失败记录。
- 用户名错误时仍执行一次 Argon2id 密码校验，避免通过响应时间判断用户名是否正确。
- 客户端地址只接受 Uvicorn 直接连接地址，或来自 `BLREC_FORWARDED_ALLOW_IPS` 中可信代理的转发地址。删除当前对任意 `X-Forwarded-For` 的信任，防止伪造 IP 绕过限速。
- 不增加全局锁定，避免外部请求把管理员本人一起锁死；不增加验证码、短信、TOTP、硬件密钥或 WAF。

## 数据与错误处理

认证数据库继续只保存密码哈希、会话/CSRF 哈希、限速状态和审计事件，不保存用户名、API Key 或明文密码。限速键带操作范围前缀，例如登录与初始化恢复分别计数。服务重启后锁定状态仍保留。

环境变量中的用户名格式无效，或群晖 Docker 未提供 API Key 时，服务启动应给出明确配置错误；HTTP 响应不得回显配置值。前端收到 `429` 时显示剩余等待时间，其他认证失败使用统一文案。

## 验证

后端测试覆盖首次设置三项校验、日常登录不需要 API Key、恢复时双凭据校验、用户名错误的等价响应、两组限速相互隔离、锁定持久化以及可信/伪造转发地址。Angular 测试覆盖三种表单字段、自动填充属性、请求体和 `429` 提示。最后运行完整 pytest、Angular 测试、静态检查、生产构建，并在本机浏览器完成设置、退出、登录和密码恢复冒烟。
