# 主列表布局统一设计

## 目标

以片段管理为视觉基准，统一房间管理、录制任务、上传任务和片段管理四个主列表的可用宽度、页面留白与列表间距，并修复片段管理内容超出视口后无法纵向滚动的问题。

同时消除录制任务和上传任务在实时状态变化后反复进入整表加载的问题。操作完成后允许数据在后台收敛，但页面已有内容不得被清空或反复显示加载状态。

## 范围

- 四个主列表都使用侧栏以外的完整可用宽度，不再受 `1180px` 最大宽度限制。
- 页面标题继续使用现有标题组件；标题下方的筛选区、列表和分页统一采用左右 `24px`、底部 `24px` 的内容留白。
- 筛选区维持 `12px` 内边距和 `12px` 下间距，列表继续使用现有中等行距，不改变列内容、响应式折叠规则或业务操作。
- 房间管理保留现有自定义表格结构，只调整外层容器与留白，使其与另外三个列表对齐。
- 片段管理组件占满主内容高度，并由页面自身承担纵向滚动；窄屏时左右留白收敛为 `12px`。
- 上传进度的标量变化继续直接更新当前行；状态变化需要校准操作项时使用不遮挡内容的后台刷新。
- 实时快照新增任务或当前页任务消失时允许后台同步列表；分页外任务退出五分钟实时窗口时不得触发当前页刷新。
- 房间管理继续直接应用实时房间快照，片段管理继续在操作成功后主动刷新一次；两者不增加新的轮询或实时订阅。

## 不在范围内

投稿账号、网络管理、设置和通知页面不调整；不改接口、筛选逻辑、分页逻辑和列表字段。

## 验证

- 桌面宽度下，四个主列表左右边界和标题/筛选/列表间距一致。
- 片段数量超过一屏时可滚动到底部并操作分页。
- 窄屏布局仍可用，现有列表响应式规则不退化。
- 删除多个上传任务后，列表内容保持可见，不因每个后台状态变化反复闪烁；分页外任务退出实时快照不会请求当前页。
- 相关 Angular 组件测试、代码检查和生产构建通过。
