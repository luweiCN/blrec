# 录制房间投稿设置设计

## 目标与范围

将投稿规则从独立页面迁入“录制任务”中的房间卡片。每张卡片增加“投稿设置”操作，点击后只加载该房间的规则并打开弹窗。独立“投稿规则”导航和页面删除，旧 `/upload-policies` 地址重定向到 `/tasks`。

本期完善现有投稿链路已经能够提交的稳定字段：账号策略、规则开关、标题/简介/分 P/粉丝动态模板、分区、标签、稿件类型、转载来源、可见范围、动态发布、转载声明、评论/弹幕设置，以及审核通过后的 SC/上舰索引评论和弹幕回灌。加入合集、自定义封面、定时发布记录为后续功能，本期不实现。上传线路继续由上传协议自动选择；文件移动/删除和高能剪辑不属于投稿设置。

## 页面与交互

侧栏“任务”改名为“录制任务”。房间卡片的第五个操作为“投稿设置”，弹窗标题显示主播名和房间号。弹窗宽度允许桌面端完整显示，但表单始终采用单列分组布局，避免窄数字输入框和宽标签输入框并排失衡。

表单分为四组：

1. **规则与账号**：启用规则、跟随主账号/固定账号；
2. **稿件信息**：标题模板、简介模板、分 P 标题模板、分区、标签、自制/转载、转载来源；
3. **发布与互动**：公开/仅自己可见、投稿后发布动态、粉丝动态模板、未经允许禁止转载、精选评论、关闭评论、关闭弹幕；
4. **审核通过后**：发布 SC/上舰索引评论、回灌视频弹幕。

新建规则默认启用；公开可见、投稿后发布动态、禁止转载、自动索引评论和弹幕回灌默认开启；精选评论、关闭评论和关闭弹幕默认关闭。粉丝动态模板仅在“投稿后发布动态”开启时编辑和提交。既有规则保持数据库中的原值，不通过迁移修改用户选择。

开启自动索引评论时禁止同时关闭评论；开启弹幕回灌时禁止同时关闭弹幕。冲突在前端即时提示，后端仍执行同样校验。仅自己可见不自动改动后处理开关；真实接口若拒绝对应操作，任务按现有错误状态暂停并展示原因。

## 动态分区

前端不再允许新规则直接填写分区数字，而是显示可搜索的一级/二级级联选择器，同时在选项中保留分区 ID 和说明。后端使用规则实际解析到的投稿账号 Cookie 请求 B 站投稿预处理接口 `/x/vupre/web/archive/pre`，避免浏览器跨域和凭据暴露。

分区结果按账号 ID 和凭据版本缓存 24 小时。打开弹窗优先使用有效缓存；缓存过期时至多发起一次刷新；上游失败时返回最后一次成功结果并标记为过期。弹窗提供手动刷新按钮。若从未成功获取分区，新规则不能保存；已有规则仍显示已保存的分区 ID，但不静默替换。

## 数据与投稿协议

`room_upload_policies` 增加以下字段，并写入创建上传任务时的不可变策略快照：

- `part_title_template`
- `dynamic_template`
- `is_only_self`
- `publish_dynamic`
- `no_reprint`
- `up_selection_reply`
- `up_close_reply`
- `up_close_danmu`

投稿载荷继续使用现有 `/x/vu/app/add` 和 TV token 签名，不切换协议。`publish_dynamic=false` 映射为不推送空间动态；为 `true` 时提交渲染后的动态文案。其他布尔值直接映射到 APP 投稿字段。分 P 标题由模板按每个 part 的上下文分别渲染，不再固定为 `P1`、`P2`。

策略接口新增按房间读取和按账号读取分区树的端点。接口只返回展示所需的分区名称、ID、父级、说明、缓存时间和是否过期，不返回上游原始响应或任何账号凭据。

## 验证与兼容

后端测试覆盖迁移、默认值、字段往返、冲突校验、策略快照、投稿载荷、分 P 标题和分区缓存/降级。前端测试覆盖卡片入口、弹窗默认值、单房间加载、级联分区、动态开关联动、冲突提示、导航删除及旧路由跳转。最后运行 Python 投稿与 Web 路由测试、Angular 无头测试、lint 和生产构建。
