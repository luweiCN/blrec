# 单时间轴高光剪辑设计

## 目标

高光剪辑页只保留一个与视频严格同步的自定义时间轴。播放跳转、高光定位、片段起止点、秒级微调、命名和创建都在时间轴区域原地完成。页面不再要求用户在播放器、片段编辑器、高光列表和待创建列表之间来回移动。

本轮同时修复浏览器插件选择“收录并投稿”后，录制任务错误显示“本场不上传”的问题，并移除录像缩略图功能。

## 投稿状态修复

插件已经正确启用房间投稿规则。错误来自录制列表仍使用旧的 `uploadIntent` 字段显示当前场次意图，而新场次实际由 `uploadDecision` 和当前房间投稿规则共同决定。

后端返回的场次投稿意图应按以下顺序动态计算：

1. 本场明确设为不上传或存在抑制记录：不上传；
2. 本场明确设为上传：上传；
3. 本场跟随房间且房间投稿规则已启用：自动上传；
4. 其他情况：不上传。

录制列表只展示该计算结果，不再直接解释旧数据库字段。浏览器插件选择“收录并投稿”后，当前直播应立即显示“本场结束后上传”。

## 唯一时间轴

视频元素关闭原生控制栏，避免出现第二条进度条。自定义时间轴紧贴视频，并在同一区域内提供播放/暂停、当前时间、总时长、音量和全屏控制。

时间轴包含以下层级，但它们属于同一条时间轴，不是多个独立进度条：

- 播放头：始终反映视频的 `currentTime`；
- 高光标记：显示在对应时间位置；
- 待创建片段：可以同时保存多个蓝色范围，但只允许一个范围处于选中编辑状态；
- 已创建片段：以范围标记显示，点击后定位和高亮；
- 录制安全尾部：不可裁剪区域使用非交互状态显示。

时间轴是播放位置的唯一真值来源。播放事件更新播放头，点击或拖动时间轴则写回视频播放位置，两者不得维护独立进度。

## 指针与播放交互

- 鼠标悬浮只显示参考线和时间，不修改播放位置，也不创建编辑状态。
- 点击空白位置会暂停视频、移动播放头，并在该位置弹出“设为开始 / 设为结束”。
- 拖动播放头时暂停视频；只在松开后显示操作，不在移动过程中追随弹窗。
- 正常播放时，时间轴内部保留固定的“设为开始 / 设为结束”按钮。点击后记录当时的播放位置，视频继续播放。
- 不提供开始或结束快捷键。
- 点击高光标记会暂停并精确跳到高光时间，然后在标记旁显示相同操作。

交互命中顺序固定为：边界手柄、高光标记、已创建片段、播放头、时间轴空白。上层对象处理事件后不得触发下层跳转，避免一次操作产生两个结果。

## 待创建与已创建状态

页面可以保存多个待创建片段，但始终最多只有一个片段处于选中编辑状态。开始点和结束点可以分别设置；两者均存在且开始早于结束时，形成一个蓝色待创建范围。

- 蓝色表示已经选好大致范围、尚未创建的片段，绿色表示已经创建完成的片段；
- 当前选中的蓝色范围显示开始、结束手柄和名称弹窗，其他蓝色范围只保留范围标记；
- 点击或拖动到其他位置会收起弹窗并取消当前选中状态，但保留蓝色范围；
- 取消选中后再次设置开始和结束，会增加新的蓝色待创建片段；
- 点击任意蓝色范围会重新选中它，并恢复名称、开始和结束位置；
- 再次选择“设为开始”或“设为结束”只替换对应边界；
- 无效边界不覆盖上一次有效值，并在原地提示原因；
- 开始和结束手柄各自提供 `−1 秒 / +1 秒`，调整后视频立即定位到该边界画面；
- 有效范围形成后，在范围附近显示名称输入框、“创建片段”和“取消本段”；
- 关闭名称输入但未创建时保留蓝色范围；“取消本段”删除该蓝色范围；
- 创建期间锁定当前范围，成功后该范围从蓝色变为绿色，并加入底部状态列表。

点击已创建片段只选择、定位并高亮它，不直接修改原文件。需要调整时，以该范围复制出唯一编辑片段，确认后生成新的片段。

## 高光与片段列表

独立高光列表、独立片段编辑器和待创建片段列表全部移除。高光的名称和时间通过时间轴标记的悬浮或点击浮层展示；重命名和删除也从该浮层进入。

底部只保留“已创建片段”状态列表，用于查看处理进度、失败原因、预览、下载、投稿和删除。边界选择及微调不在该列表中完成。

## 移除缩略图

前端不再请求或展示时间轴缩略图。后端缩略图接口、生成器、缓存和相关测试一并删除，避免数 GB 录像触发昂贵的 FFmpeg 随机取帧。时间轴使用纯色轨道、刻度和业务标记表达位置。

## 验证

后端测试覆盖插件启用投稿规则后，进行中场次被计算为自动上传，以及明确上传、明确跳过和跟随已关闭规则四种情况。

前端测试覆盖：页面没有原生进度条和缩略图请求；播放与时间轴双向同步；悬浮不改变状态；点击和拖动只在结束后弹出操作；高光位于时间轴；多个蓝色待创建范围可以共存但只能选中一个；边界秒级微调定位视频；点击其他位置收起弹窗但不丢失蓝色范围；创建成功后范围由蓝色变为绿色；各层命中不会串行触发。

浏览器插件现有测试继续验证“收录并投稿”发送 `upload=true`，NAS 验收再确认进行中场次显示“本场结束后上传”。
