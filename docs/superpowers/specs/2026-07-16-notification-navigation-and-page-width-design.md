# 通知独立导航与页面宽度统一设计

## 目标

把通知设置从系统设置顶部 Tab 中移出，作为侧边主导航中的独立一级页面；同时统一所有一级页面的内容边界，消除设置页过窄、网络页定宽、任务页铺满屏幕造成的跳动感。

## 导航与路由

侧边导航顺序固定为：录制任务、上传任务、投稿账号、网络管理、设置、通知设置、关于。“通知设置”使用铃铛图标并指向 `/notifications`。

新增独立的通知功能模块，负责通知首页、六个渠道配置页及其解析器。现有通知组件文件可以保留原目录，避免无意义的批量移动，但只能由通知模块声明。系统设置模块删除通知声明、通知路由和 `NzTabsModule` 依赖。

通知首页为 `/notifications`，渠道详情继续使用现有语义，例如 `/notifications/email-notification`。原 `/settings/*-notification` 地址改为绝对重定向到新地址，已有书签不会进入 404。系统设置页仍保留 Webhook 设置，因为它是应用回调配置，不属于这次运行事件通知中心的六个通知渠道。

## 页面结构

系统设置页删除“系统设置 / 通知设置”顶部 Tab，直接显示原有系统设置内容。通知独立页增加标准页面标题“通知设置”，其下保留通知渠道卡片和事件通知规则矩阵。渠道详情页继续使用现有子页面返回与编辑交互，但返回目标改为 `/notifications`。

设置页只请求系统设置字段；通知页只请求六个渠道设置和运行事件路由，避免两个页面互相加载无关数据。

## 统一宽度

在共享布局样式中定义唯一的一级页面内容容器：宽度 `100%`、最大宽度 `1180px`、水平居中。录制任务、上传任务、投稿账号、网络管理、设置、通知设置和关于都使用该容器，不再在各页面重复写 `680px`、`1180px` 或无限宽规则。

页面滚动层统一保留稳定的左右内边距；小于容器宽度时自动占满可用空间。任务表格和通知规则矩阵如果低于自身最小可读宽度，只在各自的表格区域横向滚动，不能撑宽整个页面。抽屉、弹窗和视频播放器不受一级页面最大宽度影响。

## 响应式与无障碍

窄屏下内容容器保持 `100%`，按钮栏允许换行，现有表格最小列宽不被压缩。侧边导航链接保持真实路由链接、当前路由高亮和现有键盘行为。新通知导航使用文字标签和图标，折叠状态下保留 Tooltip。

## 验证

Angular 测试覆盖侧边导航顺序、新旧路由、设置页不再出现 Tab、通知首页与渠道链接、两个解析器的字段范围。页面组件测试验证共享容器类存在，完整测试和生产构建必须通过。最后用实际浏览器依次打开七个一级页面，在同一窗口宽度下确认左右边界一致，并检查设置、通知矩阵和任务表格没有页面级横向滚动条。
