# 单管理员会话认证设计

## 目标

用一个管理员密码替代浏览器长期保存 API Key 的方式。系统只有一个管理员，不引入用户名、角色、注册、找回密码或多用户权限。现有 `BLREC_API_KEY` 仅用于首次初始化和本机恢复，不再授权日常 API 请求。

## 初始化与登录

管理员尚未初始化时，页面只显示初始化表单。用户提交现有 API Key 和新密码；后端恒定时间校验 API Key，使用 Argon2id 保存密码哈希并立即创建会话。未配置 API Key 时，只允许从回环地址初始化。管理员已经存在后，初始化接口永久关闭。

登录只提交密码。失败按客户端 IP 和全局窗口限速，响应不区分“管理员不存在”和“密码错误”。登录成功创建 256 位随机会话 ID，数据库只保存会话 ID 哈希、CSRF 令牌哈希、创建时间、最近使用时间和过期时间。会话 30 天过期，活动时滑动续期；修改密码和 API Key 恢复会撤销所有旧会话。

## Cookie 与 CSRF

浏览器只接收 `blrec_session` Cookie，属性为 `HttpOnly`、`SameSite=Lax`、`Path=/`；请求通过 HTTPS 时增加 `Secure`。前端不再把认证信息写入 `localStorage`，也不再弹出 API Key 输入框。

`GET /api/v1/auth/session` 返回登录状态和当前 CSRF 令牌。所有 POST、PUT、PATCH、DELETE 请求必须同时拥有有效会话 Cookie 和 `X-CSRF-Token`。登录、初始化和退出接口执行严格的同源校验。页面刷新后重新读取会话和 CSRF，不要求重新登录。

## 路由、媒体和 WebSocket

`/api/v1/auth/status`、`/setup`、`/login` 是公开端点，其余管理 API 默认要求会话。媒体播放器继续使用短时签名 URL，但签名密钥改为认证数据库中的持久随机密钥，不依赖 API Key；令牌仍绑定分 P、快照和过期时间。

两个管理 WebSocket 在握手前校验会话 Cookie 和 Origin，仅接受同源或明确允许的本地开发 Origin。认证失败直接关闭，不先 `accept()`。响应统一加入禁止嗅探、禁止嵌入和安全引用来源头。

## 存储与恢复

认证数据库放在设置文件同目录的 `auth.sqlite3`，权限为 `0600`，与投稿数据库生命周期解耦。首次启动自动创建模式和媒体签名密钥。使用当前 API Key 的恢复接口只允许重设密码，不读取旧密码；恢复成功后撤销全部会话并记录安全日志，日志不包含密码、API Key、Cookie 或 CSRF。

## 验证

后端覆盖初始化一次性、Argon2id 哈希、登录限速、会话过期/续期/撤销、CSRF、同源、媒体签名和 WebSocket 拒绝测试。前端覆盖初始化页、登录页、刷新保留会话、401 返回登录页、无本地密钥以及退出。浏览器验收确认网络请求不再包含 `X-API-KEY`，存储中没有凭据。
