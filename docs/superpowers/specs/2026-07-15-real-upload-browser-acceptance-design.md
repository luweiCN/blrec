# 真实投稿与浏览器验收设计

## 目标

使用现有主账号和一段短录播，跑通生产上传链路，并在 B 站创作中心逐项核对投稿设置。本轮只补齐真实投稿暴露的缺口，不扩展合集、封面或定时发布。

## 方案选择

采用“生产链路 + 隔离验收场次”。直接脚本调用上传接口会绕过任务创建、状态入库和前端页面；等待新直播自然结束又无法控制时间与文件大小。隔离场次可以用小文件验证与正式录播完全相同的任务链路。

## 验收数据与安全边界

- 投稿账号固定为当前主账号“虚荣-卢伟”，任务创建后锁定账号 ID 和凭据版本，不回退、不轮换。
- 复制现有 17 秒、约 9.84 MiB 的 FLV 到临时验收目录，不修改原录播文件和原场次。
- 验收场次使用当前仍在录制的太浦教育房间。该房间原场次处于 `open`，因此只有新建的短验收场次符合任务创建条件。
- 开启全局自动上传前，必须查询并断言候选场次只有验收场次。任务快照冻结后立即禁用房间规则，验收结束后关闭全局自动上传，防止其他历史场次被意外投稿。
- 首稿设为仅自己可见，关闭发布动态、自动评论和弹幕回灌。稿件保留给用户统一删除。

## 投稿配置

验收规则使用可在浏览器中明确辨认的值：

- 标题：`[BLREC投稿验收] {{ title }}`
- 简介：`BLREC 自动投稿验收\n主播：{{ anchor_name }}\n直播间：https://live.bilibili.com/{{ room_id }}`
- 分 P 标题：`P{{ part_index }}-浏览器验收`
- 标签：`BLREC测试,直播回放,{{ anchor_name }}`
- 分区：从主账号实时目录中选择 ID `21`
- 创作声明：“内容无需标注”，不勾选“未经作者允许，禁止转载”
- 仅自己可见：开；发布动态：关
- 精选评论、关闭评论、关闭弹幕：关

规则先通过本地 Angular 页面保存，以同时验证表单、动态目录和后端约束。任务创建后以 `policy_snapshot_json` 为唯一投稿依据，后续修改房间规则不影响已创建任务。

## 上传与状态流转

正式运行时的数据流为：

1. `UploadCoordinator.create_ready_jobs()` 从已归集场次和启用规则生成不可变任务快照。
2. `UposUploader` 创建预上传会话，按块上传并持久化 ETag，完成后写入远端 filename。
3. `UploadCoordinator` 使用任务绑定账号的 Web Cookie 和 CSRF 调用 `/x/vu/web/add/v3`，确认响应后写入 AID/BVID。
4. `ReviewWatcher` 在创作中心列表中核对稿件身份，并按远端 filename 一一映射分 P CID。
5. 前端上传任务页展示上传、投稿、审核和 CID 状态。

只有“请求确认未发出”才可自动重试。投稿结果未知时先从创作中心对账，不盲目重投；账号失效则暂停任务并保留原绑定账号。

## 浏览器验收

优先使用 `agent-browser` 的独立无头会话，不连接用户正在使用的 Chrome。会话仅在内存或临时目录中注入已保存的 B 站 Web Cookie，不输出凭据、不写入仓库。若创作中心明确拒绝无头浏览器，再回退到可见浏览器。

浏览器分两端验收：

- 本地 `http://127.0.0.1:4200`：保存规则，刷新后核对回填；查看任务创建、分片上传、AID/BVID 和 CID 状态。
- B 站创作中心：按 BVID 打开稿件编辑或详情页，核对标题、简介、分 P 标题、分区、标签、创作声明、仅自己可见和动态开关。

页面截图、无隐私的结构化验收结果和浏览器日志放在 `/tmp/blrec-upload-acceptance/`，不进入 Git。

## 任务文件路径复制

上传任务抽屉中的录制、成品和弹幕文件仍只显示省略后的文件名，悬浮文件名显示完整路径。每个存在的路径右侧增加一个紧凑的复制图标按钮：

- 按钮悬浮文案为“复制完整路径”，同时提供相同的无障碍名称。
- 点击后复制原始绝对路径，文件名的省略样式和抽屉状态不变。
- 成功时显示“已复制完整路径”，失败时显示“复制失败，请重试”，不静默吞掉浏览器权限错误。
- 复制按钮是独立操作，不使用“点击文件名即复制”的隐式交互，也不为单一操作增加二级菜单。

## 测试与验收标准

任何真实验收暴露的缺口都先用失败测试固化，再做最小修复。自动测试覆盖候选场次隔离、账号快照、UPOS 断点续传、未知投稿结果对账、AID/BVID/CID 入库以及前端状态展示。

完成标准：

1. 验收任务获得真实 AID、BVID 和每个分 P 的 CID。
2. 任务从创建到审核的状态在本地页面可见，失败原因不被吞掉。
3. B 站创作中心中的所有可见投稿字段与任务快照一致。
4. 没有产生额外历史场次任务，没有发送动态、自动评论或回灌弹幕。
5. 验收后全局自动上传关闭，房间规则禁用，本地前后端保持运行。
6. 三种文件路径的复制按钮都能复制完整路径，并为成功与失败提供明确反馈。
