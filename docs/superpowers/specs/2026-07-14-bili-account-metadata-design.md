# B 站账号信息与按需续期设计

## 目标

在不改变现有 Angular 15 + RxJS + ng-zorro 前端架构的前提下，让账号卡片清楚展示头像、添加时间、凭据过期时间和凭据版本含义，并把当前“立即检查续期”的强制刷新行为改成真正的检查并按需续期。

## 数据与接口

新增数据库迁移，为 `bili_accounts` 增加非敏感字段 `avatar_url` 和 `credential_expires_at`；现有 `created_at` 直接作为账号添加时间。头像取自扫码登录身份校验接口返回的 `face`，过期时间取自加密凭据包的 `expires_at`。登录、续期或健康检查成功后更新这些元数据，账号列表不为展示而额外请求 B 站，也不需要在每次加载时解密凭据。

账号列表 API 增加 `avatarUrl`、`createdAt` 和 `credentialExpiresAt`。任何接口都不返回 token、Cookie、刷新 token 或完整上游响应。

## 续期行为

账号按钮改名为“检查并按需续期”。点击后先校验现有 TV token 和 Web Cookie；只有距离过期不足 72 小时，或 B 站返回 `refresh=true` 时，才调用刷新接口并原子替换整组凭据。无需续期时保持凭据版本不变。

`credentialVersion` 是内部并发安全版本：首次保存为 1，每次成功替换整组凭据后递增。前端继续显示该值，并通过问号提示解释它不是账号等级或软件版本。

## 界面

每个账号条目显示：头像、昵称、UID、状态、添加时间、凭据过期时间和凭据版本。头像 URL 为空或加载失败时使用昵称首字作为回退。时间按浏览器本地时区显示；未知过期时间显示“暂未获取”。不新增页面或前端框架。

## 验证

- 数据库迁移保留现有账号并加入新字段。
- 登录与续期正确保存头像和过期时间。
- 未到期检查不刷新、不增加凭据版本；到期或上游要求时刷新一次。
- API 只返回非敏感元数据。
- Angular 测试覆盖头像回退、三个时间/版本字段、提示文字和按钮行为。
