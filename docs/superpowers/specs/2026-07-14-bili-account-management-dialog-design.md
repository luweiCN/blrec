# 投稿账号管理弹窗设计

## 目标

将现有“投稿”页面明确为独立的投稿账号管理入口，并把扫码登录从常驻卡片改为按需打开的弹窗。账号列表保持页面主体，减少无关信息占用空间。

## 页面与命名

- 侧边栏入口由“投稿”改为“投稿账号”，继续使用 `/uploads` 路由，避免破坏现有地址。
- 页面标题改为“投稿账号管理”，副标题保留账号凭据与自动续期说明。
- 页面右上角显示主按钮“添加账号”。
- 账号列表改为单列全宽卡片；头像、状态、时间、凭据说明和续期按钮保持现有行为。

## 添加账号弹窗

使用现有 `UploadsComponent` 内的 `nz-modal`，复用二维码状态和轮询逻辑，不新增登录组件或状态库。

1. 点击“添加账号”打开弹窗，初始只显示“生成登录二维码”。
2. 点击生成后申请二维码，并显示等待扫码、已扫码、取消、过期及失败状态。
3. 登录成功后停止轮询、关闭弹窗、重新加载账号列表，并在页面提示“账号添加成功”。
4. 用户在等待扫码或已扫码时关闭弹窗，前端立即停止轮询，并调用现有取消接口释放服务端会话。
5. 创建二维码或轮询失败时弹窗保持打开，并提供重新生成入口。

## 变更边界

本次只调整 Angular 导航、页面布局和登录交互。后端登录接口、数据库、凭据加密和自动续期规则不变。头像继续使用 `no-referrer` 避免 B 站图片防盗链。

## 验证

组件测试覆盖入口命名、弹窗打开、二维码不会自动生成、成功后自动关闭和刷新列表、主动关闭时取消会话，以及头像和账号元数据的既有行为。最后运行上传模块测试、目录 lint 和生产构建。
