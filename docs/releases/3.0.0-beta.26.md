# BLREC 3.0.0-beta.26

这是针对本地录像连续播放、故障恢复和倍速控制的公开测试版。自动化验证已完成，但尚未完成真实环境验收。

## 连续播放与恢复

- 已确认出现一分钟后暂停的 P2 原文件包含完整的 357.934 秒 H.264/AAC 数据，时间戳连续且可跨故障点完整解码；问题位于网页播放器而非录像文件。
- FLV 点播启用 stash 缓冲，将前向缓冲窗口调整为 120 秒，并在剩余 60 秒时恢复 Range 请求，避免原先 60 秒边界附近断流。
- 自动重建播放器时不再提前消费恢复位置；等待媒体元数据可用后再跳回断点并继续播放。
- 若浏览器在索引时长之前错误触发播放结束，会保留断点并自动重开，连续失败最多重试三次后才提示人工处理。
- 大文件媒体请求显式绕过 Angular Service Worker，减少 Range 请求被中间层中断或重放的可能。
- 高光页忽略由 FLV 播放器内部卸载产生的原生媒体错误，避免误报“编码不支持”。

## 倍速控制

- 高光剪辑控制行和分 P 播放弹窗新增 0.5×、0.75×、1×、1.25×、1.5×、2× 倍速选择。
- 倍速选择保存在浏览器本机，并在两个播放器和片段预览之间复用。

## 升级说明

本版本不新增数据库迁移、环境变量或目录挂载，可从 `3.0.0-beta.25` 直接升级。升级前仍应备份 `/volume1/docker/blrec-next/config` 与当前 Compose 文件。
