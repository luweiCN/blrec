# BLREC 3.0.0-beta.29

这是针对无损高光剪辑关键帧寻址和产物校验的公开测试版。自动化验证已完成，但尚未完成真实环境验收。

## 无损剪辑寻址

- 已结束录像不再只执行一次 FFmpeg 快速 seek；系统先跳到目标关键帧前 30 秒，再顺序走到预检确定的关键帧。
- 两段式寻址兼容不同 FLV 索引行为，既不会再次向前多退一个 GOP，也不会跳到用户选择点之后的下一个关键帧。
- 截取结束位置按实际关键帧起点和用户选择终点计算，不再重复加入关键帧前导。
- 录制中录像继续采用先打开文件再定位的安全路径，历史跨分段片段也复用新的已结束录像寻址逻辑。

## 产物校验与诊断

- 视频产物允许的封装时长偏差由 2 秒收紧为 500 毫秒，避免错误片段以“成功”状态进入片段管理。
- 时长异常会显示计划时长、实际时长和具体偏差，便于区分寻址错误与真正的源录像损坏。
- 源录像存在的旧失败片段升级后可以直接重试；源录像关联已经丢失的历史记录仍会阻止无效重试。

## 升级说明

本版本不新增数据库迁移、环境变量或目录挂载，可从 `3.0.0-beta.28` 直接升级。升级前仍应备份 `/volume1/docker/blrec-next/config` 与当前 Compose 文件。
