# BLREC 3.0.0-beta.27

这是针对高光剪辑失败恢复和片段删除一致性的公开测试版。自动化验证已完成，但尚未完成真实环境验收。

## 高光剪辑恢复

- 单分 P 无损剪辑会把用户选择的起点交给 FFmpeg 快速 seek，同时保留检查阶段确定的关键帧前导和计划时长，不再重复向前退一个 GOP。
- 源录像仍然存在的时长误判任务可以直接重试；源录像关联已经丢失的旧任务会明确提示无法重试，避免无效地反复排队。

## 片段删除

- 删除接口仍采用可恢复的后台任务，但界面不再提前宣称“片段已删除”，而是显示已提交处理。
- 正在删除的记录立即从片段列表隐藏；后台删除失败时记录会重新出现，并展示失败原因和“重试删除”操作。
- 对旧版本遗留在录像目录、且视频与弹幕文件均已不存在的片段路径安全跳过；片段目录外真实存在的文件仍受路径所有权校验保护。

## 升级说明

本版本不新增数据库迁移、环境变量或目录挂载，可从 `3.0.0-beta.26` 直接升级。升级前仍应备份 `/volume1/docker/blrec-next/config` 与当前 Compose 文件。
