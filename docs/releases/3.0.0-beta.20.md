# BLREC 3.0.0-beta.20

这是以请求性能、后台任务持久化和异常恢复为重点的公开测试版。自动化验证已完成，但尚未完成真实环境验收。

## 页面与请求性能

- 房间、录制、上传和片段列表改用轻量汇总查询与按需详情，减少数据库往返、文件系统访问和前端首屏工作量。
- SSE 只采集当前页面订阅的数据，网络接口发现和上传目录读取会缓存或合并并发请求。
- 直播信息批量刷新、画质解析、直播封面和归档查询会复用同一轮结果，不提高原有房间轮询频率。

## 后台任务与异常恢复

- 设置保存与运行时应用分离，房间控制、删除、上传重试和片段检查使用可恢复的持久化任务。
- 上传协议、通知、Webhook、扫码登录和弹幕分页增加明确的并发与超时边界，避免单次异常长期占用 worker。
- 修复设置 worker 异常退出后不再处理任务或关机卡住、直播流重连继续复用失效地址等问题。

## 升级说明

本版本会自动执行数据库迁移 `0024` 至 `0029`，用于列表索引、删除恢复、片段检查和批量上传重试。升级前必须备份 `/volume1/docker/blrec-next/config` 与当前 Compose 文件；四个持久化目录和环境变量没有变化，可从 `3.0.0-beta.19` 直接升级。
