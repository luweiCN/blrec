# BLREC 3.0.0-beta.32

这是面向直播与片段集中管理的公开测试版。本次新增永久收藏媒体库、外部媒体导入、按直播场次聚合的片段管理，以及两处片段重命名入口。beta.31 因发布质量门禁未通过而没有生成公开镜像，本版本包含其全部功能和质量修正。

## 永久收藏与媒体库

- 录制任务可直接收藏整场直播。系统会创建媒体库条目，保留原直播、房间、分 P、弹幕、投稿任务和 B 站稿件关联。
- 收藏文件会移动到与 `/rec` 同级的 `/favorites` 目录，并原子更新所有数据库路径；跨文件系统移动会先复制、校验，再切换引用。
- 收藏直播和收藏片段只支持手动删除，不计入录像容量，不参与保留期、低空间回收或上传完成后的自动删除。
- 媒体库支持预览、重命名、标签、剪辑、重新投稿、查看投稿记录和删除；一场直播可继续按多个分 P 管理。

## 外部上传与片段管理

- 可从浏览器分批上传系统外采集的完整直播或独立片段，并纳入同一套媒体库管理。
- 外部直播同样支持名称、标识、分 P、剪辑和投稿；外部片段按永久内容处理。
- 片段管理页改为按来源直播场次聚合，无来源直播的片段保持为独立分组。
- 剪辑页底部的“已创建片段”和独立片段管理页都可重命名片段，名称会同步用于展示与下载文件名。

## 升级说明

本版本包含数据库迁移 `0030_initial.sql`，应用启动时会自动执行。升级前请备份 `/volume1/docker/blrec-next/config` 和当前 Compose/环境配置。

必须使用本版本的 `compose.synology.yml`，并创建 `/volume1/docker/blrec-next/favorites`，确保它挂载到容器 `/favorites`。若只修改镜像标签而没有新增该挂载，永久收藏文件会落入容器可写层，重建容器后可能丢失。

可从 `3.0.0-beta.30` 直接升级；原 `/rec`、`/clips`、配置、日志和投稿状态均会保留。
