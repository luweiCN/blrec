# BLREC 3.0.0-beta.33

这是修正媒体库实际操作体验的公开测试版。本版本统一了直播收藏与片段管理入口，并修复收藏录像预览和分 P 剪辑入口的问题。

## 预览与剪辑

- 收藏直播的录像通常仍为 FLV。媒体库预览现在复用录制任务播放器的 FLV/MSE 适配，不再把 FLV 签名地址直接交给浏览器原生播放器。
- 外部导入的 MP4 等浏览器原生格式继续直接播放，收藏直播与外部导入直播使用一致的预览交互。
- 多分 P 直播的剪辑入口移动到每个具体分 P；进入剪辑器时会携带对应分 P 标识，不再提示必须从录制任务详情进入。

## 统一媒体库

- 媒体库增加“直播收藏”和“片段”两个页签。
- “片段”页签同时管理从直播剪出的片段和外部导入片段，原独立片段管理入口不再占用主导航。
- “上传片段”统一改称“导入外部片段”，明确它指从系统外采集并永久保存的视频，不是系统剪辑生成的片段。
- 直播剪辑片段继续支持按来源场次聚合、预览、重命名、投稿和删除；外部导入片段继续使用媒体库的重命名、标签、投稿和手动删除能力。

## 升级说明

本版本不新增数据库迁移、环境变量或目录挂载，可直接从 `3.0.0-beta.32` 升级。

升级前仍应备份 `/volume1/docker/blrec-next/config` 和当前 Compose/环境配置。原 `/rec`、`/favorites`、`/clips`、配置、日志和投稿状态均会保留。
